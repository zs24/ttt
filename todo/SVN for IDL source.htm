SVN for IDL source

Description:

Currently we have about 1500 IDL source files in /pet/source/idl/. They are important to our daily operation, image analysis, etc. Now they are not in any version control, and everyone can modify these. This can bring problems that we don’t know which version was used for a specific study and user can make a mistake to modify these.

Goal:

1.      Put the whole folder and its contents into SVN version control so that we can track changes and which version is used for specific study.

2.      Implement access control in the folder and its contents so that only privileged users can make modification.

3.      Implement access control to the SVN repository of IDL source codes.

Challenge:

1.      Users are used to working directly in this folder directly. They will feel surprised and frustrated if they lose access to this folder.

2.      Users will have to create their own local IDL folder and work on it. To make it work, he/she must set environment variables so that local IDL folder can be found. This is not easy to everyone.

3.      When users want to upload their IDL program from local folder to SVN server, they must know how to use the SVN commands. This can become a big issue to most users.

4.      The folder /pet/source/idl/ will become our common working folder. How will it be synchronized with SVN server and checkout new changes? Here, most users don’t have access and cannot do this synchronization.

Solution:

1.      Make the folder /pet/source/idl and all its contents owned by hrrt (or another pseudo user), and remove the group write permission for the folder and its contents. In this case, only user hrrt can make changes.

2.      Nobody should directly work in this common folder.

3.      Only a few (about 5-6) users can access SVN repository. Each user in the 5-6 users will be responsible to a category of source files. For example, JD will in charge of modeling program, Tim will be in charge of recon files, etc. Each user will take care of modify source code in SVN repository, commit, and update in common source code.

4.      Each user will work in their local IDL folder. If he/she wants to commit their work to SVN repository and common IDL folder, he/she needs to contact responsible person to do this.

5.      Create more detailed instruction on how to use SVN commands to work with local IDL folder and how to checkout from/commit to SVN server. I’ll be happy to help everyone (only 5-6 users, not 20-30 users) in the next few months (or years).
